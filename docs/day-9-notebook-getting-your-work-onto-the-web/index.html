Day 9 · Getting your work onto the web

Measuring Manuscripts

Durable publishing comes down to one idea: static files. A picture, a table, a page: nothing that has to keep running on a server. This notebook turns your analysis into those files and bundles them for your GitHub Pages repository.

1. Save a figure¶

Whatever your result is, save the picture of it as a PNG. Replace the placeholder plot with your own.

import matplotlib.pyplot as plt
fig, ax = plt.subplots(figsize=(6, 4))
ax.bar(['a', 'b', 'c'], [3, 7, 4])   # === TO BUILD: your real figure ===
ax.set_title('My result')
fig.savefig('figure.png', dpi=150, bbox_inches='tight')
print('Saved figure.png')

2. Save a data table¶

CSV so others can reuse the numbers, HTML so they render on the page.

import pandas as pd
df = pd.DataFrame({'item': ['a', 'b', 'c'], 'count': [3, 7, 4]})   # === TO BUILD: your data ===
df.to_csv('data.csv', index=False)
df.to_html('table.html', index=False, border=0)
print('Saved data.csv and table.html')
df

2b. A data dictionary¶

A CSV is only reusable if a stranger knows what each column means. Write that down once, as a small metadata file, and ship it with the data.

# Describe each column so the CSV is reusable by someone who didn't make it.
data_dictionary = {
    'item':  '=== TO BUILD: what this column holds ===',
    'count': '=== TO BUILD: what this column holds, and its units ===',
}
with open('data_dictionary.txt', 'w') as f:
    f.write('Data dictionary for data.csv\n')
    f.write('============================\n\n')
    for col in df.columns:
        desc = data_dictionary.get(col, '(undocumented)')
        f.write(f'{col}: {desc}\n')
print('Saved data_dictionary.txt')
print(open('data_dictionary.txt').read())

3. An interactive chart (optional)¶

A static PNG is the durable default. For a chart readers can hover over, Plotly writes a self-contained HTML file, still a static file, no server.

try:
    import plotly.express as px
    fig2 = px.bar(df, x='item', y='count', title='My result (interactive)')
    fig2.write_html('chart.html', include_plotlyjs='cdn')
    print('Saved chart.html')
except ImportError:
    print('Plotly not installed. Run  !pip install plotly  if you want this. Optional.')

4. The page itself¶

A web page is just a text file. Here is a small, complete one with your title, a sentence, the figure, and the table.

html = '''<!doctype html>
<html lang="en"><head><meta charset="utf-8">
<title>TO BUILD: project title</title>
<style>body{font-family:system-ui,sans-serif;max-width:760px;margin:40px auto;padding:0 16px;line-height:1.5}
img{max-width:100%}table{border-collapse:collapse}td,th{padding:4px 10px;border-bottom:1px solid #ddd}</style>
</head><body>
  <h1>TO BUILD: project title</h1>
  <p>TO BUILD: one paragraph saying what this is and what you found.</p>
  <img src="figure.png" alt="result">
  <h2>Data</h2>
  ''' + open('table.html').read() + '''
  <p><a href="data.csv">Download the data (CSV)</a></p>
</body></html>'''
open('index.html', 'w').write(html)
print('Saved index.html')

5. A README¶

The file that lets a stranger (including your future self) understand and rebuild the project. It’s the single thing that most often keeps a project alive.

readme = '''# TO BUILD: project title

One paragraph: what this is, who made it, and what it shows.

## Data
Where the data came from, and its format.

## How to rebuild
Open the notebook in Colab and run all cells.

## License
See LICENSE.
'''
open('README.md', 'w').write(readme)
print('Saved README.md')

6. A license¶

Without a license, no one legally knows what they may do with your work. CC BY 4.0 is a common, permissive choice for scholarship. It asks only for credit.

license_note = ('This work is licensed under CC BY 4.0.\n'
                'https://creativecommons.org/licenses/by/4.0/\n'
                '=== TO BUILD: add your name and year ===')
open('LICENSE', 'w').write(license_note)
print('Saved LICENSE')

6b. A citation file¶

If you archive a frozen snapshot (e.g. on Zenodo), it gets a DOI: a permanent address others can cite. A CITATION.cff file tells people exactly how to cite your work. Fill in your details. Add the DOI once you have one.

# A CITATION.cff file: standard, machine-readable, recognised by GitHub and Zenodo.
citation = '''cff-version: 1.2.0
message: "If you use this work, please cite it as below."
title: "TO BUILD: project title"
authors:
  - family-names: "TO BUILD: surname"
    given-names: "TO BUILD: given name"
date-released: "2026-07-01"
license: "CC-BY-4.0"
# Add this line once you have archived a snapshot on Zenodo:
# doi: "10.5281/zenodo.XXXXXXX"
'''
open('CITATION.cff', 'w').write(citation)
print('Saved CITATION.cff')
print(citation)

7. Bundle it for download¶

Zip the whole site so you can download it in one click, then drop the files into your repository.

import zipfile, os
files = [f for f in ['index.html','figure.png','table.html','data.csv','chart.html','README.md','LICENSE'] if os.path.exists(f)]
with zipfile.ZipFile('my_project_site.zip', 'w') as z:
    for f in files:
        z.write(f)
print('Bundled:', files)
try:
    from google.colab import files as colab_files
    colab_files.download('my_project_site.zip')
except Exception:
    print('(In Colab this would download. Use the Files panel on the left to grab the zip.)')

7b. Check your bundle¶

Before you publish, list what’s in the folder and flag anything missing. A project without a README or a license is a project that won’t last.

# List the files you've produced and flag the ones that keep a project alive.
import os
produced = sorted(f for f in os.listdir('.')
                  if os.path.isfile(f) and not f.endswith('.ipynb'))
print('Files in this folder:')
for f in produced:
    print('  -', f)

print()
essentials = {
    'index.html': 'the page itself',
    'README.md': 'so a stranger can rebuild it',
    'LICENSE': 'so reuse is legal',
}
missing = [name for name in essentials if not os.path.exists(name)]
if missing:
    print('MISSING - add these before publishing:')
    for name in missing:
        print(f'  - {name}  ({essentials[name]})')
else:
    print('All essentials present: index.html, README.md, LICENSE. Ready to publish.')

🔧 Then make a GitHub repository, upload these files, and enable Pages (Settings → Pages). Your project is then live at a URL you can put on a CV. The Git/GitHub handout covers the steps.
